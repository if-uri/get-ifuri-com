if get.ifuri.com

Uruchom komputer node przez jeden URI-ready installer.

Ten endpoint instaluje `urirun` z GitHuba, generuje lokalne bindings i registry, a potem startuje `urirun node`, ktory host moze odkryc i wywolywac przez URI flow.

Default node port 8765, bind 0.0.0.0, registry w ~/.urirun-node

Install + run

Na komputerze node uruchom w foreground:

curl -fsSL https://get.ifuri.com/node.sh | bash

Background node

Instalacja z nazwą i portem, po czym proces zostaje w tle:

curl -fsSL https://get.ifuri.com/node.sh | bash -s -- --name laptop --port 8765 --background

Host registration

Na komputerze host dodaj node do mesh:

urirun host init --name host
urirun host add-node laptop http://NODE_IP:8765
urirun host nodes
urirun host routes
urirun host agents

NL to URI flow

Host moze wygenerowac i wykonac flow po dostepnych trasach URI:

urirun host ask "sprawdz stan laptopa i zapisz notatke" --execute
1. Install Tworzy venv i instaluje `urirun` z `tellmesh/urirun`.
2. Registry Generuje bindings v2 i kompiluje lokalne URI registry.
3. Serve Wystawia `/routes`, `/mcp/tools`, `/a2a/card` i `/run`.
